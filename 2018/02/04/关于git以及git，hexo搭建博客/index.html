关于git以及git与hexo结合搭建博客

首先确定已经安装git

一、创建版本库：
① midir 文件名1
cd 文件名1
pwd //显示文件所在目录
② git init //将该目录变成git管理的仓库
③ 将要 写的文件2放到之前创的文件名的目录下（打开notepad然后另存为）
④ git add 文件名2 //把文件添加到仓库 （add可以多次提交不同文件，中间没有符号）
⑤ git commit -m”wrote a 文件名2 file” //把文件提交到仓库
二、
① git status //用来查看结果（是否被修改）
② git diff //用来查看做了哪些修改
③ 再提交 add 、 commit
④ git log //查看“从终至始”都做了哪些修改
⑤ git reset –hard HEAD^(上一个版本) HEAD^^(上上个版本) HEAD~100（上一百个版本）
git reset –hard commit_id （指定到的版本）
cat 文件名2 //查看当前
⑥ 当改乱了时，想丢弃工作区的修改， git checkout –file
三、将本地库传到github上：
① git remote add 名字 git@github.com:4melogo/github库名.git
② git push -u 名字 master

四、从远程库克隆：
① 在github上新建一个库
② git clone git@github.com:4melogo/库名.git
③ 从远程库拉取 git pull 库名 master:master

五、分支：
① 查看分支 git branch
② 创建分支 git branch 分支名
③ 切换分支 git checkout 分支名
④ 创建并切换分支 git checkout -b 分支名
⑤ 合并某分支到当前分支 git merge 分支名
⑥ 删除分支 git branch -d分支名 删除远程分支 git push 库名 –delete 分支名
⑦ 在分支里插入 vi+文件名2
⑧ 按:wq退出插入
（1）冲突：
① 先提交新分支，再切换到master提交。再合并。
② 创建分支时，可以使用–no-ff方式的git merge
先创建分支，然后git add文件名2 ，然后git commit -m” “,，然后切回master分支，合并时 git merge –no-ff-m” “分支名
（2）bug分支：（当在其他分支dev上的工作没有完成，但是要修复bug时，用 git stash储藏起来）
假设在master上出现bug，就在master上创建分支修复，修复完成后，再切换到master分支，并合并，再删除修复分支
（如果还要继续在没完成的工作的分支dev上工作，就切换到dev分支，用git stash list查看，用git stash pop恢复并删除stash内容）
六、多人协作：
① 查看远程库信息 git remote -v
② 从本地推送分支，使用git push 远程库名 分支名，如果推送失败，先用git pull抓取远程的新提交；
七、标签管理：
① 创建标签：先切换到需要打标签的分支上，然后git tag 标签名（可以用git tag查看所有标签）
如果要给以前的commit id打标签。可以先git log –pretty=oneline –abbrev-commit，找到要打的id，然后git tag 标签名 commit id
② 查看标签信息 git show 标签名
③ 创建带有说明的标签 git tag -a 标签名 -m” “id，用git show 标签名查看说明文字
④ 私钥标签 git tag -s 标签名 -m” “id
⑤删除标签 git tag -d 标签名
⑥ 推送标签到远程 git push origin 标签名 如果一次性推送全部标签：git push origin –tags
⑦删除远程标签： 先删除本地标签，再git push origin：refs/tags/标签名

八、自定义git
① git config –global color.ui true 显示颜色

搭建博客

——在博客上发表文章：打开cmd， hexo new “文章名”，在自己的文件里，source里的posts会出现 文章名.md。进行编写，之后hexo g， hexo d
